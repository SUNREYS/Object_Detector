## Critical Fix: Stop Converting Images to Grayscale

### The problem
load_image_grayscale() (evaluate.py:182) forces all images to single-channel.
But ALL images (PID, visible, thermal, PI-GAN) are 3-channel RGB (512x512x3, uint8).
The YOLO detector sees RGB. Converting to grayscale:
- Throws away color information the detector uses
- Makes LPIPS impossible (requires RGB)
- Loses per-channel quality differences (e.g. correct luminance but wrong colors)
- Hides color-specific artifacts/ghosts

### The fix
- Primary loading: load_image(path) -> returns BGR as-is (3-channel)
- Grayscale only when a specific metric needs it (Laplacian sharpness, Canny edges),
  done internally via cv2.cvtColor(img, cv2.COLOR_BGR2GRAY) inside that metric's function
- SSIM, LPIPS, ghost detection: operate on the RGB image (or per-channel)
- Per-object quality features: brightness = mean across all channels, contrast = std
  across all channels, edge strength = computed on luminance channel internally

### eval_metrics.py (quality metric computation)
What: Computes image quality features that measure how faithful the synthetic images are
and how that affects detection. Operates on RGB images (not grayscale).

- load_image(path) -> 3-channel BGR uint8 as-is (NO grayscale conversion)
- load_thermal_gt(flat_name, split) -> 3-channel thermal GT image
- compute_object_quality_features(img_bgr, x1, y1, x2, y2, ...) -> per-object dict
  - brightness: mean pixel intensity across all 3 channels -> brightness = mean(crop) / 255
  - contrast: std of pixel intensities across all channels -> contrast = std(crop) / 255
  - fg_bg_diff: |mean(foreground) - mean(background border)|
  - edge_strength: Sobel magnitude on luminance (internal grayscale conversion)
  - blur/sharpness: Laplacian variance on luminance (internal grayscale conversion)

  Comment style example:
    # -- Brightness ------------------------------------------------
    # Mean pixel intensity across all 3 RGB channels.
    # brightness = (1/N) * sum(pixel_values)  where N = H * W * 3
    # Scaled to [0, 1] by dividing by 255.
    # High brightness -> well-lit scene; low -> dark/underexposed.

- compute_ghost_features(gen_crop_bgr, thermal_crop_bgr) -> per-object dict
  - object_ssim: SSIM between crops (computed per-channel, then averaged)
  - ghost_score: structured residual (Gaussian-blurred |gen - thermal|)
  - hallucination_score: edge pixels in generated not in thermal

- NEW: LPIPSScorer class

  LPIPS (Learned Perceptual Image Patch Similarity)
  -------------------------------------------------
  Measures perceptual distance using deep features from AlexNet.

  Math:  d(x, y) = sum_l  w_l * || normalize(f_l(x)) - normalize(f_l(y)) ||^2
  where: f_l = features at layer l of pretrained AlexNet
         w_l = learned weights calibrated against human perceptual judgments
         normalize = unit-length normalization in channel dimension

  Range: 0 = perceptually identical, ~0.5+ = very different

  Why for this thesis:
    SSIM measures structural similarity (pixel-level statistics).
    LPIPS measures whether the image LOOKS right to a human.
    A generated pedestrian might have correct brightness/edge stats (good SSIM)
    but unrealistic textures or colors (bad LPIPS). If LPIPS correlates with
    detection failure, it means the detector is sensitive to perceptual quality
    -- the pedestrian needs to "look real" to be detected.

  - Loads AlexNet model once (__init__)
  - score(gen_crop_rgb, thermal_crop_rgb) -> float (per-object)
  - Resizes crops to 64x64, normalizes to [-1, 1] tensors
  - Only computable for synthetic modalities (PID, PI-GAN), not visible

- NEW: FIDComputer class

  FID (Frechet Inception Distance)
  ---------------------------------
  Measures distance between two IMAGE DISTRIBUTIONS (not individual images).
  One FID number per modality = "how realistic is this generator overall?"

  Math:  FID = ||mu_gen - mu_real||^2 + Tr(Sigma_gen + Sigma_real - 2 * sqrtm(Sigma_gen @ Sigma_real))
  where: mu, Sigma = mean and covariance of 2048-dim InceptionV3 pool3 features
         sqrtm = matrix square root (scipy.linalg.sqrtm)

  Lower FID = distribution of generated images is closer to reference (thermal).

  Why for this thesis:
    Unlike per-image metrics, FID captures whether the MODEL as a whole
    produces realistic images. A model could have some good and some bad
    images (per-image SSIM varies), but FID tells us if the overall
    distribution is right. Comparing FID vs mAP answers: does a more
    realistic generator produce a better detector?

  - Uses torchvision InceptionV3 (already installed, no new dependency)
  - extract_features(image_dir, ...) -> Nx2048 feature matrix
  - compute(gen_dir, real_dir) -> FID float
  - Also computes FID_day and FID_night subsets

- NEW: compute_sharpness_size_table(df_obj):

  Cross-tabulates recall by sharpness level x object size.

  sharpness = Var(Laplacian(crop))  -- higher = sharper
  Objects split at MEDIAN sharpness into "low"/"high".
  Sizes from YOLO bbox height: small (<=32px), medium (<=96px), large (>96px).

  Output: 2x3 table (+ day/night variants):
                  Small    Medium    Large
  Low sharpness   recall   recall    recall   (n=...)
  High sharpness  recall   recall    recall   (n=...)

  Thesis question: Do small objects need high sharpness to be detected?
  Hypothesis: small objects have few pixels -> blur destroys features ->
  sharpness matters more for small than large objects.

- NEW: analyze_false_positives(df_fp, df_img):

  False Positive Analysis: WHY does the detector hallucinate?
  ----------------------------------------------------------

  1. Confidence distribution of FPs:
     - Low conf (0.25-0.5): less worrying, detector is uncertain
     - High conf (>0.5): detector is strongly fooled
     high_conf_fp_rate = count(conf > 0.5) / total_fp

  2. Image conditions that produce FPs:
     - Pearson r(fp_count_per_image, ghost_score per_image)
     - Pearson r(fp_count_per_image, hallucination_score)
     - Day vs Night FP rate: fp_rate = FP / (TP + FP)

  3. What the detector "sees" at FP locations:
     - Extract brightness, contrast, edge_strength at each FP box
     - Compare TP crops vs FP crops: are FP locations brighter? More edges?
     - This answers: are FPs triggered by ghost artifacts, shadows, or textures?

  4. Per-FP quality features (computed on the FP bounding box):
     - fp_brightness, fp_contrast, fp_edge_strength, fp_blur
     - If thermal GT available: fp_ghost_score, fp_hallucination_score
       (Is the FP region where the generation produced artifacts?)

- NEW: root_cause_analysis(df_obj):
  Uses sklearn LogisticRegression to predict detected (True/False) from
  [brightness, contrast, fg_bg_diff, edge_strength, blur, object_ssim,
   ghost_score, hallucination_score, bbox_area, day_night].
  Reports standardized coefficients as feature importance.

## Comment/Documentation Style

Every module gets a top-level docstring explaining WHAT and WHY.
Section headers use banner comments:

  # =============================================================================
  # 3A. SSIM VS DETECTION PERFORMANCE
  #   Measures whether structural similarity to thermal GT predicts detection
  #   quality. Uses Pearson correlation at image level.
  # =============================================================================

Math formulas before the code:

  # -- Ghost Score -----------------------------------------------
  # Measures structured artifacts in generated images vs thermal GT.
  # 1. Compute pixel-wise absolute residual: R(x,y) = |gen(x,y) - thermal(x,y)|
  # 2. Gaussian blur to suppress random noise: S = GaussianBlur(R, ksize)
  # 3. ghost_score = mean(S), range [0, 1]
  # High ghost_score -> large structured differences -> visible artifacts

No comments on obvious code. Focus on WHY and MATH.
